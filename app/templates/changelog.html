Recent updates and improvements to the MAGFest Budget System.

{# March 31 2026 - Nav polish, override rework, account type overrides #}

March 31, 2026

Improved Updated Navigation Bar

The top navigation bar has been visually refreshed with a cleaner, more polished look.

New Event Override: Account Type

Expense account event overrides now support changing the Account Type per event. For example, an account that is "Standard" by default can be set to "Fixed Cost" for a specific event, and it will correctly appear in the Fixed Costs tab for that event's budget requests.

Improved Redesigned Event Override Form

The expense account event override page has been rewritten for clarity. It now uses the same Account Type dropdown as the base form, includes the Default Spend Type and Frequency fields, and has help text explaining what each setting does and how it relates to the base account.

Fixed Department Visibility in Fixed Cost / Hotel / Badge Tabs

Expense accounts restricted to specific departments now correctly respect those restrictions across all tabs, including Fixed Costs, Hotel, and Badges.

Improved Help Text on Admin Forms

The base expense account form and event override form now include more descriptive help text to explain what each field controls.

{# Production Launch - Late March 2026 #} {# Production Launch - Late March/Early April 2026 #}

Launch - April 2, 2026

New Welcome to the MAGFest Budget System!

The budget system is now live for all staff. Department heads can create and submit budget requests, reviewer groups will receive email notifications when items are dispatched for review, and budget admins can track everything through the approval workflow. If you run into issues, please reach out to #money-questions-and-discussions on Slack or accounting@magfest.org.

Improved Faster Dashboards

The home page, division view, and admin request list now load significantly faster, especially for users with access to many departments.

Improved More Reliable Workflow

Submitting, dispatching, and finalizing budget requests will now always succeed, even if email notifications are temporarily unavailable. Admins can no longer accidentally finalize a request while a reviewer is actively working on it.

Improved Security Hardening

Various security improvements including safer handling of links in descriptions and redirect validation.

The MAGFest Budget System is now publicly available on GitHub. If you're interested in contributing or reporting bugs, check out the repo. We've added contributor guidelines, security reporting, and automated testing to support community contributions.

{# Late March 2026 #}

Late March 2026 ▼

New Persistent Navigation Bar

A new top navigation bar is now visible on every page, inspired by the RAMS registration system. Role-based dropdown menus give you quick access from any page: Budget Admin (workflow queues, expense accounts, reviewer groups), Reports (ledger, department summary, workload, missing budgets), Review (your reviewer group queues), Admin (system config for super admins), and a user menu showing your divisions and departments for the current event.

New Division Landing Page

Division heads can now click their division name in the user menu to see all departments in that division for the current event, with budget status at a glance.

Improved Admin Dashboards

The System Admin page now shows a system overview with counts for users, divisions, departments, active events, and reviewer groups. The Budget Admin page gained a new event progress section showing how many departments have started budgets and a request status breakdown. Both pages had redundant link sections removed since the navigation bar now handles quick access.

Improved Cleaner Home Page

Removed the Budget Admin Quick Actions and Quick Links sections from the home page since all those links are now available in the persistent navigation bar. The home page now focuses on your department overview, key dates, and how-it-works guides.

Improved Meaningful Request IDs

Budget requests now have human-readable IDs like SMF27-TECHOPS-BUD-1 instead of random codes. The format includes the event, department, work type, and a sequential number. Both primary and supplementary requests share the same sequence, so you might see SMF27-PANELS-BUD-1 (primary), SMF27-PANELS-BUD-2 (supplementary), etc. Existing requests keep their original IDs.

Improved Protected Event and Department Codes

Event cycle and department codes are now locked once budget requests exist. This prevents accidental changes that would break request IDs and URLs. The admin edit forms clearly indicate when a code is locked and why.

Improved Supplementary Request UX

Supplementary requests now have better identification and context. When creating a supplementary, you can optionally add a reason (e.g., "Additional equipment needed" or "Revised vendor quote"). In list views, supplementary requests now show as "Supplementary #1", "Supplementary #2", etc., along with the creation date and reason. The reason can be edited while the request is still in draft status.

New Badges Tab

The budget draft screen now has a dedicated "Badges" tab for requesting staff, contractor, guest, and attendee badges. Badge requests are informational only (no budget impact) and help track badge needs across departments.

New Admin-Editable Tab Content

Budget admins can now customize the help text and descriptions shown in budget tabs (Budget Lines, Fixed Costs, Hotel, Badges, Notes). Access this from Admin Settings > Site Content. Each section can display as plain text or a highlighted info box.

New Report Filtering by Request Type

The Department Report and Ledger Report now include a "Request Type" filter. You can view all requests, primary only, or supplementary only. This filter applies to both the on-screen display and CSV exports.

New Experimental Priority Level

Added a new priority option: "Experimental / Stretch Goal" for non-operational, exploratory, or aspirational expenses. Use this for items that would be nice to pursue if budget allows but aren't essential for operations.

Improved Required Fields on Budget Lines

Priority, Confidence, and Frequency are now required fields when adding budget lines. This ensures reviewers have the context they need to make informed recommendations.

{# Mid-March 2026 #}

Mid-March 2026 ▼

New Editable Email Templates

Budget admins can now edit email notification templates directly from the admin interface. Customize subject lines and body text for submission, dispatch, needs attention, response received, and finalized notifications. Includes a live preview, test email feature, and basic formatting toolbar for bold, italic, underline, and links.

New Event Organization Dashboard

Super admins can now manage which divisions and departments participate in each event from a single page. Access it from Admin Settings or the Quick Links on the homepage. Disabled divisions are collapsed by default and visually distinguished.

Improved User Management Terminology

Changed "Members" to "Users" throughout the admin pages to reduce confusion. Departments are still "members" of divisions (org structure), but people assigned to divisions/departments are now called "users."

New What's New Page

Added this changelog page so everyone can see recent updates. Find it in the user menu at the top right.

New Quick Access to Event Organization

Added a direct link to the current event's organization settings in the Admin Settings page.

{# end changelog-content #}
{# Early March 2026 #}

Early March 2026 ▼

New Clickable Links in Expense Account Descriptions

URLs and markdown-style links in expense account descriptions are now clickable. This makes it easy to link to policy documents, Google Docs, or other resources directly from expense account descriptions.

New Per-Event Expense Account Descriptions

Expense accounts can now have different descriptions for different events. For example, the "Hotel Suites" account can show MAGWest-specific information when working on MAGWest budgets, and different info for Super MAGFest.

Improved Faster Page Loading

Optimized database queries throughout the system for faster page loads, especially on pages showing many budget lines or departments.

Improved More Reliable Email Notifications

Improved error handling for email notifications. If something goes wrong, the system now logs details to help troubleshoot delivery issues.

Improved Hotel Suite Options Updated

Updated hotel suite options to match official policy terminology.
